# spec.md - 기능 명세서

## 1. 프로젝트 개요

사진과 짧은 메모를 입력하면 AI(Claude API)가 블로그 글을 자동 생성해주는 웹 앱.
특정 플랫폼(네이버) 전용이 아닌, **범용 블로그 글 생성기**.

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend/Backend | Next.js (React) + API Routes |
| AI | Claude API (Anthropic) - 이미지 + 텍스트 멀티모달 |
| 배포 | Vercel |

## 3. 페이지 구성

### 3-1. API 키 입력 (첫 화면)
- Claude API 키 입력 필드
- 입력 즉시 유효성 검증 (잘못된 키/잔액 부족 시 안내 메시지)
- "Claude API 키 발급 방법" 가이드 링크 제공
- Claude API는 구독(claude.ai)과 별개의 종량제 서비스임을 안내
- API 크레딧 충전이 필요하다는 안내 (console.anthropic.com 링크)
- 인증 성공 시 글 작성 페이지로 이동
- API 키는 sessionStorage에 저장 (새로고침 유지, 탭 닫으면 삭제)

### 3-2. 글 작성 (메인 페이지)
- 한 화면에 섹션별로 구분하여 표시
- 상세 구성은 아래 "4. 입력" 참조

### 3-3. 사용 가이드
- 앱 사용 방법 안내 페이지

## 4. 입력

한 화면 내에서 **섹션 구분**(카드/구분선)으로 나뉨.

### 섹션 1: 사진 & 메모

#### 사진 업로드
- 최대 30장 (유연하게 조정 가능한 숫자)
- 지원 포맷: jpg, png, webp
- 드래그 앤 드롭 업로드
- 업로드 순서 = 글의 흐름 (스토리 순서)
- 드래그로 순서 변경 가능
- 개별 삭제 가능
- 사진 없이 메모만으로도 생성 가능 (대신 상세한 메모 필요)
- 클라이언트에서 긴 변 기준 1568px 자동 리사이즈 + base64 변환
- 사진 속 텍스트(간판, 메뉴판 등) AI가 읽어서 활용

#### 메모
- **전체 주제 메모** (필수) - 비어있으면 다음 단계 진행 불가, 안내 메시지 표시
- **사진별 개별 메모** (선택) - 각 사진 아래에 메모 입력칸, 안 쓰고 넘어갈 수 있음

### 섹션 2: 설정

#### 카테고리 (5개 고정)
- 맛집
- 일상
- 여행
- 제품리뷰
- 홍보

#### 톤/말투 (매번 사용자가 선택)
- 프리셋 5개:
  - 음슴체 (~했음, ~먹음)
  - 해요체 (~했어요, ~좋아요)
  - 합쇼체 (~했습니다, ~좋습니다)
  - 반말 (~했어, ~맛있었어)
  - 친근한 존댓말 (~했는데요, ~거든요)
- 자유 입력 (직접 설명 가능)

#### 글 길이
- 짧게 (약 500자 내외) / 보통 (약 1000~1500자) / 길게 (약 2000~3000자)

#### 언어 선택
- 기본값: 한국어
- 사용자가 원하는 언어로 변경 가능

#### 검색 키워드 (선택)
- 사용자가 입력한 키워드를 AI가 글에 자연스럽게 녹여서 작성

#### 사용자 자유 서술 (선택)
- 추가 요청사항을 자유롭게 작성 가능
- 예: "사진 설명은 짧게 해줘", "소제목 넣어줘"

### 생성 버튼
- 생성 전 **예상 비용 안내** (사진 장수 × 토큰 수 기반 대략적 비용)
- 전체 주제 메모 필수 유효성 검사 통과 후 생성 가능

## 5. 글 생성

### 배치 처리
- 사진 10장씩 배치로 Claude API에 전달
- 1~10장 처리 → 결과 요약 → 요약을 다음 배치에 전달 → 11~20장 처리 → 반복
- 글 흐름 유지 + 토큰 한도 초과 방지

### 스트리밍 응답
- ChatGPT처럼 글자가 실시간으로 출력되는 방식
- Vercel 타임아웃 우회 + 빠른 UX 제공
- 최대한 빠른 속도로 출력

### 로딩 화면
- DINO 로고 알파벳이 하나씩 점프하는 애니메이션

### 생성 중 이탈 방지
- 뒤로 가기/새로고침 시 "생성 중입니다. 나가시겠습니까?" 경고 팝업

### 결과물 구조
- **제목**
- **본문** (사진 순서대로 스토리 구성)
- **해시태그** (10개 미만, AI 판단)

### 본문 세부사항
- 인트로/마무리 멘트: AI가 알아서 자연스럽게 추가
- 소제목: 사용자가 요청할 때만 삽입
- 부적절한 콘텐츠 방지: Claude API에 위임

## 6. 프롬프트 관리

### 구조
최종 프롬프트 = 카테고리 프롬프트 + 설정값 + 사용자 자유 서술

1. **카테고리별 기본 프롬프트** - 별도 파일로 분리 (예: `prompts/restaurant.txt`)
   - 맛집: 맛 표현, 식감, 가격, 위치 중심
   - 여행: 감성, 풍경, 코스 중심
   - 일상: 일기 느낌, 감정 중심
   - 제품리뷰: 장단점, 사용 경험 중심
   - 홍보: 특장점, 소구 포인트 중심
2. **설정값 반영** - 톤/글 길이/언어/검색 키워드
3. **사용자 자유 서술** - 추가 요청사항 반영

### 프롬프트 파일 분리 이유
- 글 퀄리티 튜닝 시 프롬프트만 수정하면 됨 (코드 수정 불필요)
- 카테고리 추가 시 파일 하나만 추가하면 확장 가능

## 7. 편집

### 에디터
- **사진 있을 때**: 사진 블록 단위 에디터 (사진 아래 텍스트를 각각 편집)
- **사진 없을 때**: 일반 텍스트 에디터로 전환

### 재생성
- 사용자가 피드백을 주면서 재생성 가능 (예: "이 부분 좀 더 자세하게")
- AI가 부족한 정보 있으면 사용자에게 역질문 가능
- 재생성 시 이전 버전은 덮어쓰기
- 대화 턴 수 제한 없음 (사용자 API 비용)
- 대화 히스토리는 세션 메모리에서 관리

## 8. 미리보기

- 범용 블로그 스타일 레이아웃
- 구조: 제목 → 사진 + 사진 아래 글 → 본문 → 해시태그
- 특정 플랫폼에 종속되지 않는 깔끔한 디자인

## 9. 복사

- 미리보기 화면에서 **전체 복사** 버튼
- **HTML 복사** (기본) - 서식 유지
- **텍스트 복사** (옵션) - 원하면 텍스트로도 가능
- 사진 미포함 - 사진 위치에 `[사진 1 삽입]` 같은 placeholder 표시
- 복사 대상: 제목 + 본문 + 해시태그 전체

## 10. 에러 처리

- API 키 잘못됨: 입력 시 즉시 검증 + 안내 메시지
- API 잔액 부족: 안내 메시지
- 네트워크 연결 실패: "연결할 수 없습니다" 안내
- 전체 주제 메모 미입력: 생성 버튼 비활성화 + 안내

## 11. 기타

- **PC 우선**, 모바일은 Responsive 되는 선에서 지원
- **API 키 보안**: API 호출은 Next.js API Route(서버)에서만 실행, 클라이언트에 키 노출 방지
